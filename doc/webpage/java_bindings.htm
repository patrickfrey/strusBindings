Java language bindings

Introduction

The language bindings have been built using SWIG based on a wrapper C++ interface. This wrapper interface serves as common base for all language bindings as a sort of super interface (strus/bindingObjects.hpp).

Doxygen interface documentation

There exists a documentation for the Java interface generated with doxygen. You will also find there some examples to introduce the Java bindings.

Description

All objects of the strus Java API are in the namespace net.strus.api. The first thing you do, when using the strus Java interface, is to create the root object called StrusContext. From this context object you can construct the main instances of the objects doing the job you want to do. Depending on how you create the strus context, the objects you create beginning with the root object are accessed directly or they are residing on a server and the language objects serve as proxy for the server objects.
If you want to access the strus storage directly, you create the context without passing any argument to the constructor: 

			String storageConfig = "path=storage; metadata=doclen UINT16";
			StrusContext ctx = new StrusContext();
			StorageClient storage = ctx.createStorageClient( config);
If you want to access strus over a proxy for a strus RPC server, you pass the connection string as <address>:<port> (example "localhost:7181") as first argument of the StrusContext constructor and you create the storage client without configuration: 
			String rpcServer = "localhost:7181";
			StrusContext ctx = new StrusContext( rpcServer);
			StorageClient storage = ctx.createStorageClient("");

What is better depends on your use case. The possibility to use strus as proxy was mainly introduced for accessing strus in the context of a webserver, where holding of objects beyond the scope of a connection is difficult or impossible. An other argument is that the command line tools of strus can be used simoultaneously with some other instance like a server accessing the storage.
In case of Java, the use as proxy will be less likely. Most of the examples written for Java will therefore show the direct access to strus.
